vocabÂ¶

torchtext.experimental.vocab.vocab(ordered_dict, min_freq=1, unk_token='<unk>')[source]Â¶

Factory method for creating a vocab object which maps tokens to indices.

Note that the ordering in which key value pairs were inserted in the ordered_dict will be respected when building the vocab. Therefore if sorting by token frequency is important to the user, the ordered_dict should be created in a way to reflect this. Additionally, the if the unk_token isnâ€™t found inside of the ordered_dict, it will be added to the end of the vocab.

Parameters

  • ordered_dict (collections.OrderedDict) â€“ object holding the frequencies of each token found in the data.

  • min_freq â€“ The minimum frequency needed to include a token in the vocabulary. Values less than 1 will be set to 1. Default: 1.

  • unk_token â€“ The default unknown token to use. Default: â€˜<unk>â€™.

Raises

ValueError â€“ if a default unk_token isnâ€™t provided.

Examples

>>> from torchtext.experimental.vocab import vocab
>>> from collections import Counter, OrderedDict
>>> counter = Counter(["a", "a", "b", "b", "b"])
>>> sorted_by_freq_tuples = sorted(counter.items(), key=lambda x: x[1], reverse=True)
>>> ordered_dict = OrderedDict(sorted_by_freq_tuples)
>>> v1 = vocab(ordered_dict)
>>> tokens = ['e', 'd', 'c', 'b', 'a']
>>> v2 = vocab(OrderedDict([(token, 1) for token in tokens]))
